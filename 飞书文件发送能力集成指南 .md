# 飞书文件发送能力集成指南 
 
本文档用于在本地 OpenClaw 环境中启用和验证飞书文件发送功能。 
 
--- 
 
## 一、前置条件 
 
### 1.1 飞书应用权限 
 
确保你的飞书开放平台应用已开通以下权限： 
 
- **im:message** - 发送消息权限 
- **im:message.file** - 发送文件消息权限（关键） 
- **im:resource** - 资源上传权限（用于上传文件到飞书） 
 
**检查方式：** 
1. 登录 [飞书开放平台](https://open.feishu.cn/) 
2. 进入你的应用 → 权限管理 
3. 确认上述权限已添加并发布 
 
### 1.2 飞书插件配置 
 
OpenClaw 的飞书插件需要正确配置凭证： 
 



bash
检查飞书插件是否已安装
openclaw config get plugins --json


应包含 `feishu` 插件条目。

bash

### 2.2 工具权限



bash
检查是否有工具被禁用
openclaw config get tools.deny --json


**注意：** 如果 `message` 工具在 deny 列表中，需要移除：



bash
获取当前 deny 列表（记录输出）
openclaw config get tools.deny --json
重新设置不含 "message" 的数组（根据实际输出调整）
openclaw config set tools.deny '["web_search","web_fetch"]' --strict-json


### 2.3 飞书插件工具状态



bash
检查飞书 IM 相关工具是否可用
openclaw config get skills.entries.feishu-im-user-message.enabled


如返回 `false` 或未设置，启用它：



bash
openclaw config set 'skills.entries.feishu-im-user-message.enabled' true

## 三、自检流程
### 3.1 快速自检脚本
将以下内容保存为 `check-feishu-file.sh` 并执行：



bash
#!/bin/bash

echo "=== 飞书文件发送能力自检 ==="
echo ""
1. 检查飞书插件
echo "1. 检查飞书插件配置..."
FEISHU_CONFIG=$(openclaw config get channels.feishu --json 2>/dev/null)
if [ $? -eq 0 ] && [ -n "$FEISHU_CONFIG" ]; then
echo " ✓ 飞书通道已配置"
else
echo " ✗ 飞书通道未配置，请先配置飞书插件"
exit 1
fi
2. 检查工具限制
echo "2. 检查工具限制列表..."
DENY_LIST=$(openclaw config get tools.deny --json 2>/dev/null)
if echo "$DENY_LIST" | grep -q '"message"'; then
echo " ✗ message 工具在 deny 列表中，需要移除"
echo " 执行：openclaw config set tools.deny '[...]' --strict-json"
else
echo " ✓ message 工具可用"
fi
3. 检查飞书技能
echo "3. 检查飞书 IM 技能..."
IM_ENABLED=$(openclaw config get 'skills.entries.feishu-im-user-message.enabled' 2>/dev/null)
if [ "$IM_ENABLED" = "true" ]; then
echo " ✓ feishu-im-user-message 技能已启用"
else
echo " ⚠ feishu-im-user-message 技能未启用或不存在"
echo " 执行：openclaw config set 'skills.entries.feishu-im-user-message.enabled' true"
fi
4. 检查 Gateway 状态
echo "4. 检查 Gateway 运行状态..."
if openclaw gateway status 2>/dev/null | grep -q "running"; then
echo " ✓ Gateway 运行中"
else
echo " ⚠ Gateway 未运行，请先启动服务"
echo " 执行：sh scripts/start.sh"
fi

echo ""
echo "=== 自检完成 ==="


**使用方法：**



bash
chmod +x check-feishu-file.sh
./check-feishu-file.sh

## 四、常见问题排查
### 4.1 错误：无法发送文件消息
**可能原因：**
- 飞书应用缺少 `im:message.file` 权限
- 文件大小超过限制（飞书限制 100MB）
- 文件路径不存在或不可读
**解决方案：**
1. 检查飞书开放平台权限配置
2. 确认文件大小 < 100MB
3. 使用绝对路径，并确认文件存在：



bash
ls -lh /path/to/your/file.md


### 4.2 错误：凭证失效
**症状：** 所有飞书操作失败，返回认证错误
**解决方案：**



bash
撤销授权后重新授权

在飞书对话中对 Agent 说："撤销授权"
然后重新触发授权流程
### 4.3 错误：Gateway 未响应
**症状：** 配置命令无响应或超时
**解决方案：**



bash
重启 Gateway
sh scripts/restart.sh
查看日志
tail -f ~/.openclaw/logs/gateway.log
